一个在网络上的家
Aug 12 2020

第一次建属于自己的个人博客网站，挺开心的，也算是在网络上有了一个属于自己的家。

这个网站是借助 github pages 和 Hexo 而建成的，在建站过程中看到一篇介绍特别详细的 博客教程 ，如果有建立个人博客的想法的同学可以参考这篇博客。

建这个这个博客的时候还踩了点雷。一开始想用clover这个主题的，但是执行hexo s就一直报错，后来看到网上也有用clover却不显示的，就用vs code看了下，clover的源程序报错，然后改用了white。

然后就是封面和图床，封面的设置在网上找到了个说是photo，但是没用，就去问了原作者，属性是cover。图床的话，希望 picgo 稍微优化下，最后选择用的是百度到的 路过图床 ，希望这个图床活得久一点。

2021年12月31日 | github在国内总是随缘登陆，近期发现网易uu加速器可以免费加速github等学术资源。（刚上传博客时，uu加速器弹出的学术资源窗口里没github，而前几天都是排在热门资源第一位，以为是没法加速与github的连接了，但试了下上传，还是可以连接到github的）

我在修改博客的时候，有时候不做标记和记录，有时候会在修改的地方加一个括号，写着修改时间，也有文末加分割线与修改时间，也有近期想出来的增加博客记录分段。

博客的内容会有时间先后顺序的排列（编写时间或事情发生的时间），也有关联关系的排列，愈发感受到写作的学问之多。

2022年1月24日 | 换了新电脑（联想拯救者R9000P2021），也是将本地博客挪到新电脑上，以下几行则是之前尝试出的一个比较快速的转移本地博客方法：

①下载git、nodejs

②安装Hexo（npm install -g hexo-cli）

③创建本地博客（hexo init myBlog||cd myBlog||npm install）

④将该新建本地博客theme文件夹、source文件夹、_config文件替换为旧电脑本地博客的文件（可执行hexo s测试本地博客是否正常运行）

⑤配置SSH KEY

⑥在本地博客文件下安装部署插件（npm install hexo-deployer-git –save）并完成上传（hexo g -d）

2022年2月21日 | 这些天想了一种博客定稿的方式，可以将本地文件设为只读，刚也用空气渲染式动圈结构耳机实用新型专利那篇博客试了效果，在设置为只读后再在vs code里编辑是无法保存的，因此今天会将一些无后续更新的博客的本地文件设置为只读，如果有必须要编辑的部分则可以取消本地文件的只读设置，而设置为只读、取消只读设置的这种博客修改是不需要再更新的博客不可避免的操作，需要博客记录分段的记录，而此后需要更新的博客又都会使用博客记录分段，因此会在所有博客里添加博客记录分段（这篇hello-world除外）。

但将本地文件设置为只读并不等于定稿，正如创业的逐步成型那篇博客中关于定稿的内容所说——“但博客只要是还能修改，就没什么真正意义上的定稿”。

2022年3月14日 | 博客的更新逐渐会给自己带来些焦虑，特别是在去年那次意识流记录前后记录风格的一些改变后，例如会担心有的更新是不是有漏记录到博客记录中的（hello-world博客除外），而在此之前更多时间是在思考创业的逐步成型内容该怎么写，还未成型时，下一步又该做什么调整。更早的时候，初高中的时候，还有过花费很多时间去想软件和硬件的相关设计。

还有类似于意识流那段内容里写到的，在一篇博客中，为了避免内容的臃肿，会在写新的内容时寻找是否已有高度相似的内容，如果已有高度相似的内容，则不会再过多赘述。

对于带链接的文字段，最开始会在前后加空格，但不加空格隔开也还不错，所以博客会存在这两种写法。

在本地文件设置为只读后，如果在Windows环境下的VS Code里再修改保存，会提供Overwrite、Save as、Discard三个选项，选择Discard即可清除修改，而修改后直接关闭文件会提供Save、Don’t save、Cancel三个选项，选择Don’t save即可清除修改。

（从自2022年起的创业新阶段博客移至hello-world博客，2022年3月9日）以前博客更多追求简洁，读者能更快读懂，而在创业的逐步成型那篇博客记录风格的改变和意识流记录后，博客更多追求记录的全面，尤其是创业相关的这些内容。

2022年3月15日 | 写的东西，有的是为了记录，有的是为了向别人传递些信息，有的也会兼具这两个目的，想清楚这点对写不偏科这件事来说很重要，而且对其他一些事情来说也很重要。

由于不会在一篇博客中重复写高度相似的内容，所以先写一些涵盖范围广的内容，也可以实现控制篇幅的效果。

博客里，除了数字的特定使用（主要是品牌名称），其他地方的数字的形式都会较为随意（如时间），汉字形式和阿拉伯数字形式都会使用。

无论是博客还是日记，对当天内容的修改都会很轻松，而对于之前内容的修改则会感觉是对之前那个日子的侵犯。

与昨天提及的带链接的文字段前后是否加空格、今天提及的数字形式一致，提及别的博客文章等网页链接时，本博客内的文章有时带链接，有时不带链接，主要时指代容易不清的会带链接，整体比较随意，本博客外的一般都会带链接。类似这样相同情况的不同写法的随意使用有很多，主要是不太想给自己太多限制。

这篇博客开头的github pages带链接的文字段写成了github pages和hexo，之前已经几次忘记修改了，今天终于修改了。

这几天hello-world这边有点像当时对意识流的记录，大致就是大脑里围绕某件事想到很多（hello-world和意识流那边主要都是围绕写博客这件事），然后判断想出来的是有意义的，还是没太多意义的，最后去记录那些有意义的，而这个记录的过程对我来说是会占用时间与精力的，希望少一些围绕写博客这件事的，多一些围绕各类产品、软硬件开发这类事情的（这里原打算写围绕研究，但思考如何写博客也算一种研究，所以换成更准确的用词）。

写博客的过程也在帮助自己更好的用词、更好的语言组织。

2022年3月16日（这边日期一开始都写漏了） | 不太方便写一些东西的时候，可以采用现在手机便签里记下几个关键词，等方便写的时候再扩写。

这几天日记写的有点多，博客里也有些其他内容的更新，但希望重心不要太偏博客，少些一些，虽然博客的记录也有很大的意义，但STM32和墨水屏、耳机测试、软件开发、各科学习也很重要。记录还是得有些取舍，虽然对一些细节得记录和一些对写博客得思考的记录很有意思，但不应过多占用做那些更为实质的事情的时间（写博客也是实质的事情，记录是不可或缺的，但软硬件开发等事情是更为实质的）。PS.记录的取舍也应建立在可以真实还原事情的基础上。

博客里继续保持一些细节的记录（如博客记录等），然后会有其他形式的整合，例如写成图书等。

取消博客本地文件的只读（设置和修改都比较繁琐，也会担心是否有该设置只读的设置漏了，该设置为可读可写的误被设置为只读，虽然暂时没发现这些问题，而且当初设置只读的原因可以被之后想的一些解决方法处理），均设置为可读可写，之前设置为只读也并非没有意义，至少给每篇博客（除hello-world）都加上了博客记录分段。

记录归记录，生活还是得好好生活，虽然记录成为生活的一部分时，生活的结构自然会有点改变，但要找到记录在生活结构中合适的时间占比、精力占比，找到平衡点，希望接下来会更好。具体来说就是既不至于把想的全写下来，但也不至于纠结一些可以写下来的细节该不该写（毕竟以后会有其他形式的整合），同时专注更实质的那些事情。

虽然想着少写一点，但还是会尽可能带着小方本，多记录点也不是坏事，少写些需要誊写到博客上的就行。（这段属于生活部分，但与博客相关，不至于写到创业博客里，但可以写到hello-world里）

博客里有各种类型的，例如博客本身相关的、具体产品相关的、创业记录相关的等多条记录线，所以一些内容的修改和添加可能会涉及到多篇博客文章（博客文章一般会简写为博客，虽然会跟博客网站混淆，但问题不大）。
